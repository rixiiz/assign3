1. Added move selection state
- What I did: added state to track which square is currently selected during
  the move phase
- What happened: starting from the 4th move, each move is done with selecting
  a square and a destination

2. Detect placement phase and move phase by counting player squares
- What I did: added logic to count how many X or O squares are on the board, 
  then used that to determine if the current player should still be placing 
  squares or should now be moving squares
- What happened: game can now know when players should play or move squares

3. Added adjacency check for moving squares
- What I did: added a helper function to determine whether one square is adjacent
  to another, which includes horizontal, vertical, and diagonal
- What happened: moves can now only move to an adjacent empty square

4. Implemented center square rule during move phase 
- What I did: if current player has 3 squares and one is the center one, then 
  their move must either win or move that square away
- What happened: the game now disallows moves that keep the center square 
  occupied but doesn't win

5. Removed the draw logic
- What I did: removed the old draw logic that checked whether the board was full
- What happened: chorus lapilli does not have a draw condition

6. Updated click handler in board to support placement and move phases
- What I did: changed handleClick so that when the current player has less than
  3 squares, a click behaves like before. But otherwise, the first click selects
  one of the current player's own squares, and the second click either resets
  selection on clicking on the same square again or violating the rules 
  (filled square, non adjacent square, center rule) or performs the move
- What happened: a selected square can now move to a valid adjacent empty square,
  then the turn would change

7. Updated status message for move selection
- What I did: changed the status message logic so that the UI tells the player 
  what step they are on during move phase 
- What happened: before selecting a square, it tells the user to select one, 
  and after selecting a square, it tells the user to choose an adjacent empty 
  square

8. Show selected square on board
- What I did: updated square so that a selected square is visually distinguished
  from the other squares
- What happened: made it easier for the user to tell which square was chosen
  during the first click of the move phase

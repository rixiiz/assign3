Setup:
- Started from completed tic-tac-toe implementation
- Continued using ./helper dev to check live changes

1. Added move selection state
- What I did: 
  - added new react state variables selectedSquare using useState(null)
  - passed selectedSquare and setSelectedSquare from App to Board as props
- What happened: starting from when a player has 3 squares, each move is done 
  with selecting a square and a destination

2. Detect placement phase and move phase by counting player squares
- What I did: 
  - added variables xCount and oCount inside Board
  - iterated through squares array using a loop and incremented xCount if 
    value === 'X' or incremented oCount if value === 'O'
  - determined currentPlayer using turn (true = X, false = O)
  - created boolean isMovePhase, where true if current player already has 3 
    squares on board and false otherwise
- What happened: game can now know when players should play or move squares

3. Added adjacency check for moving squares
- What I did:
  - created helper function isAdjacent(fromIndex, toIndex)
  - converted indices to row/column using row = Math.floor(index / 3) and
    col = index % 3
  - computed rowDiff and colDiff using absolute difference
  - allowed move only if rowDiff <= 1 and colDiff <= 1 and not the same square
- What happened: moves can now only move to an adjacent empty square

4. Implemented center square rule during move phase 
- What I did:
  - created helper function mustVacateOrWin(fromIndex, toIndex)
  - checked if game is in move phase 
  - check if current player occupies center square (index 4)
  - simulated move by copying squares array:
    - set fromIndex to null
    - set toIndex to currentPlayer
  - checked:
    - if simulated move results in win, which is allowed
    - if moving center square itself, which is allowed
    - otherwise, which is disallowed
- What happened: the game now disallows moves that keep the center square 
  occupied but doesn't win

5. Removed the draw logic
- What I did: 
  - removed loop that checked if board was full
  - removed isDraw variable and related status condition
- What happened: chorus lapilli does not have a draw condition

6. Updated click handler in board to support placement and move phases
- What I did:
  - modified handleClick(index) function:
    - placement phase: if square is empty, copy squares using slice(), placed 
      'X' or 'O', and called onPlay(newSquares)
    - move phase:
      - first click: if selectedSquare is null, then only allow selecting squares 
        belonging to current player and store index in selectedSquare
      - second click: 
        - if same square, cancel selection
        - if square is occupied, reset selection
        - if not adjacent, reset selection
        - if violates center rule, reset selection
        - otherwise:
          - copy board
          - move piece from selectedSquare to index
          - set selectedSquare to null
          - call onPlay(newSquares)
- What happened: a selected square can now move to a valid adjacent empty square,
  then the turn would change. Invalid moves reset selection

7. Updated status message for move selection
- What I did:
  - modified status variable logic:
    - if winner, show winner
    - if move phase:
      - if no square selected, show "select a piece"
      - if square selected, show "select an adjacent empty square"
    - otherwise, show next player
- What happened: before selecting a square, it tells the user to select one, 
  and after selecting a square, it tells the user to choose an adjacent empty 
  square

8. Show selected square on board
- What I did:
  - passed isSelected prop to Square component: 
    isSelected = (selectedSquare === index)
  - added visual styling (inline styles) to indicate selected square
- What happened: made it easier for the user to tell which square was chosen
  during the first click of the move phase

9. Improved helpful comments
- What I did: improved explaination on what each part of the code does
- What happened: code can now be understood and built on by other engineers

10. Added 3 tests to chorus lapilli

  to check that turns alternate, and test to check that clicking on an 
  occupied square won't modify its value 
- What happened: game is now more reliable and less prone to bugs
